# EcoTrack — Tech Stack

## Runtime
- **Node.js**: ≥ 20 (required by Next.js 15)
- **TypeScript**: 5.x (strict mode on)

## Frontend + Backend
- **Next.js 15** (App Router, API Routes — NOT pages/ directory)
- **React 19** (with Server Components and Client Components)
- Pattern: explicit REST API routes at `/app/api/*` — NOT Server Actions for mutations (improves testability)

## UI Libraries
- **Tailwind CSS** 3.x — utility-first styling
- **shadcn/ui** — component library (Button, Card, Input, Label, Badge, Progress, Tabs, Avatar, Toast, Skeleton)
- **Lucide React** — SVG icon library (ONLY — no emoji as icons)
- **Recharts** — charts (AreaChart for trend, PieChart/Donut for category breakdown)
- **Framer Motion** — animations (respect `useReducedMotion()` hook always)

## Typography
- **Fira Code** — headings, numbers, data labels (Google Fonts via `next/font/google`)
- **Fira Sans** — body text, labels (Google Fonts via `next/font/google`)
- `font-display: swap` on both (prevents FOIT)

## Design System
- File: `design-system/ecotrack/MASTER.md` (generated by ui-ux-pro-max)
- Style: Organic Biophilic
- Colors: Background `#020617`, Surface `#0F172A`/`#1E293B`, Accent `#22C55E`, Text `#F8FAFC`
- Rounded corners: `rounded-2xl` (16px) to `rounded-3xl` (24px)

## Database
- **Supabase** — managed PostgreSQL + Auth
- **Prisma** ORM (schema-first, type-safe queries)
- Schema file: `prisma/schema.prisma`
- Tables: users, quiz_responses, activity_logs, badges, user_badges, ai_insights
- Env vars: `DATABASE_URL` (pooler), `DIRECT_URL` (direct), `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`

## Authentication
- Supabase Auth SDK (`@supabase/ssr`)
- Browser client: `src/lib/supabase/client.ts` (`createBrowserClient`)
- Server client: `src/lib/supabase/server.ts` (`createServerClient` with cookies)
- Custom-built signup/login UI pages → calls Supabase Auth SDK
- Route protection: `src/middleware.ts` (Next.js middleware with cookie-based session check)

## AI Integration
- **Google Gemini API** (`@google/generative-ai`)
- Model: `gemini-1.5-flash`
- Env var: `GEMINI_API_KEY`
- Called ONLY from server-side API routes (key never exposed to browser)

## Testing
- **Jest** + **ts-jest** — unit + API tests
- **React Testing Library** + `@testing-library/jest-dom` — component tests
- **Playwright** — E2E tests (Chromium)
- Test directories: `__tests__/unit/`, `__tests__/api/`, `__tests__/components/`, `e2e/`
- Run unit+API tests: `npm test`
- Run E2E: `npx playwright test`

## Code Quality
- **ESLint** (Next.js config)
- TypeScript strict mode
- Import alias: `@/*` → `src/*`

## Deployment
- **Vercel** (single command: `vercel --prod` or push to main)
- Required env vars set in Vercel dashboard (same 5 as `.env.local`)
- Build command: `npm run build`

## Key Lib Files
- `src/lib/emission-factors.ts` — CO₂ emission factor constants (IPCC/DEFRA)
- `src/lib/gamification.ts` — level thresholds + badge definitions (constants only)
- `src/lib/gamification-engine.ts` — business logic (updateStreak, checkAndAwardBadges, computeLevel)
- `src/lib/co2-calculator.ts` — calculateActivityCO2, calculateBaseline
- `src/lib/offsets.ts` — static list of 6 carbon offset programs
- `src/lib/prisma.ts` — PrismaClient singleton
- `src/lib/utils.ts` — cn() helper (shadcn)
