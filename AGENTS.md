# Workspace Goal / Problem Statement

**Goal:** Design a solution that helps individuals understand, track, and reduce their carbon footprint through simple actions and personalized insights.

---

# Hackathon Constraints & Expectations

**Strict Constraint:**
- The repository size must be strictly less than 10 MB. Avoid committing large binaries or assets.

**Challenge Expectations:**
- Build a smart, dynamic assistant.
- Ensure logical decision making based on user context.
- Focus on practical and real-world usability.
- Write clean and maintainable code.

**Documentation Requirement (README):**
The README must explicitly explain:
1. The chosen vertical.
2. Approach and logic.
3. How the solution works.
4. Any assumptions made.

**Review Criteria (Always optimize for these):**
- **Code Quality:** structure, readability, maintainability.
- **Security:** safe and responsible implementation.
- **Efficiency:** optimal use of resources.
- **Testing:** validation of functionality.
- **Accessibility:** inclusive and usable design.
